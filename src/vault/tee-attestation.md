Cryptographic proof generated by a Trusted Execution Environment (Intel SGX, AWS Nitro Enclaves, AMD SEV-SNP) that specific code ran in a specific environment at a specific time.

TEE attestation is not post-hoc audit — it is execution-time accountability evidence. The agent cannot have run different code than what the attestation signed. The host operating system, hypervisor, and cloud provider cannot inspect or alter what runs inside the enclave. This answers "did this agent do what it was authorized to do?" from the attestation alone, even though no human observed the execution.

For [[agentic-sovereignty]], attestation provides the accountability bridge: non-modifiability is the privacy/integrity property, attestation is the accountability property. The same architecture that makes the agent hard to stop makes it provably honest about what code it ran. Limitation: attestation proves code identity but reveals nothing about what data was processed inside the enclave.
