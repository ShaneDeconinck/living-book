x402 is an open protocol that embeds payment directly into HTTP workflows using the long-dormant 402 "Payment Required" status code, enabling agents to pay for API calls without a human in the loop.

## How It Works

An agent makes a GET request. The server returns 402 with a JSON payload describing the payment terms (network, amount, asset, resource). The agent signs a **EIP-3009 TransferWithAuthorization** — a gasless USDC authorization using EIP-712 typed data — and replays the request with an `X-PAYMENT` header. The server verifies the signature, settles on-chain, and returns the data. The agent needs a signing key and USDC in its wallet, not ETH for gas.

The technical innovation: the agent never sends a blockchain transaction. The server settles and pays the gas. This separates authorization (agent) from execution (server), reducing the operational complexity of holding a full blockchain client.

## Why This Matters for Agents

Traditional payment rails break at agent scale:
- Credit cards require human authentication ceremonies
- Stripe's $0.30 minimum fee makes $0.01 API calls economically impossible
- Settlement latency (days) is incompatible with real-time agent workflows

x402 on Layer 2 networks solves all three. On Base: gas cost ~$0.002, settlement in seconds. A $0.01 API call becomes viable. A $0.10 query carries 2% overhead, not 300%.

## Economics by Settlement Layer

| Query Price | Base L2 (~$0.002 gas) | Ethereum mainnet (~$15 gas) |
|---|---|---|
| $0.01 | 20% overhead | 150,000% overhead |
| $0.10 | 2% | 15,000% |
| $1.00 | 0.2% | 1,500% |

Mainnet is not viable for micro-transactions. x402 adoption concentrates on L2s: Base, Solana, Polygon, Stellar, Etherlink.

## Deferred Settlement

Cloudflare proposed a deferred settlement scheme: cryptographic trust established immediately via the x402 handshake, financial settlement batched daily through traditional payment methods or stablecoins. This removes the USDC onboarding requirement — the main adoption barrier for most organizations. Makes x402 payment-rail-agnostic.

## Adoption State (March 2026)

x402 Foundation co-founded by Coinbase and Cloudflare (September 2025). Stripe integrated x402 on Base (February 2026). Cloudflare built it into Agent SDK and MCP server infrastructure. Deployed across 12+ chains.

Honest assessment: infrastructure is ahead of demand. Daily organic volume ~$28,000 with ~half estimated artificial activity (Artemis, March 2026). Transactions dropped 92% from December 2025 peak. This is typical early infrastructure adoption: major providers betting on readiness before organic use arrives.

## Payment as Trust Signal

x402 creates four trust properties beyond the transaction:
1. **Proof of authorization** — cryptographic signature proves who authorized the spend
2. **Economic accountability** — real money creates real consequences (budget alerts, anomaly detection)
3. **Sybil resistance** — creating fake agents is cheap; making them pay is not
4. **Audit trail** — on-chain settlement is immutable, compliance-grade accounting
