## 1. DIF Blog — blog.identity.foundation

**Notable developments:**

**"Why We Brought MCP-I to DIF (and Why DIF Said Yes)"** (March 5, 2026)
https://blog.identity.foundation/why-dif-said-yes-to-mcp-i/

This is the most directly actionable item in this scout run. Vouched donated MCP-I (Model Context Protocol — Identity) to DIF, where it will be developed as an open standard under a dedicated MCP-I task force inside a new Trusted AI Agents Working Group. MCP-I extends MCP with four identity dimensions: cryptographically verifiable agent identity (via DIDs), proof of principal authorization, delegated scope (verifiable credentials), and trustworthiness signaling. It defines three conformance tiers (Levels 1–3) for adoption. The article explicitly frames DIDs and VCs as the technical foundation.

**"Digital Identity for Agentic Systems"** (March 12, 2026)
https://blog.identity.foundation/digital-identity-for-agentic-systems/

A member report with enterprise use cases (insurance claims processing, regulatory compliance/fraud detection) identifying five identity requirements for autonomous agents: delegated authority, verifiable representation, policy-bound access, auditability, and revocation/accountability. Key diagnostic: current systems conflate authentication with authorization. The report cites VCs already deployed in insurance partnerships, the EUDI wallet, and vLEI as relevant infrastructure.

**"Building the Agentic Economy"** (February 20, 2026)
https://blog.identity.foundation/building-the-agentic-economy/
Part 5 of a "Building AI Trust at Scale" series — examining utility and trust mechanisms in autonomous agent ecosystems.

**Relevance:** High. MCP-I landing at DIF is a direct intersection of MCP, DIDs, VCs, and agent trust — exactly the book's terrain. The DIF enterprise report validates the book's framing that authentication/authorization conflation is a core problem.

**Actionable:** Yes. The MCP-I spec work at DIF is now a live standards process. The book should reference it. If the Trusted AI Agents Working Group is producing drafts, those are primary sources worth tracking next scout.

## 3. Google A2A — github.com/google/A2A

**Notable developments:**

**A2A v1.0.0 released March 12, 2026.**

The project reached 1.0 this week (22.6k stars, Linux Foundation backed, Apache 2.0). Key changes in v1.0.0:

- OAuth 2.0 modernization: removed implicit and password flows; added device code flow and PKCE support
- Authentication schemes now embedded in Agent Cards
- Multi-tenancy support via additional scope fields
- Breaking changes: push notification config restructured, deprecated proto fields removed, spec refactored to separate application protocol from transport mappings
- Field and ID naming standardized across protocol bindings

Planned for post-1.0: formal authorization handling, dynamic UX negotiation, expanded client-initiated methods.

**Relevance:** High. v1.0.0 is a milestone — the protocol is now stable enough to treat as a reference implementation. The OAuth modernization (removing implicit/password flows, adding PKCE) and the Agent Card authentication model are directly relevant to the book's coverage of OAuth for agents and agent-to-agent trust.

**Actionable:** Yes. The book should cite the v1.0.0 spec as the current stable reference and can now describe the authentication model with confidence it won't shift. The PKCE + device code additions align with the book's OAuth-for-agents thread.
